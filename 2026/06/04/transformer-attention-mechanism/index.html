AI 时代漫游：注意力机制

Google 在 2017 年发布了《Attention Is All You Need》，提出了大名鼎鼎的 Transformer，成为现代大语言模型的基石。而 Transformer 架构中最核心的概念叫注意力机制。

注意力到底是什么

在人类语言中，一个词语，在不同的语境中会有截然不同的含义。以“苹果”为例：

  • 句子 A：“这个苹果真好吃，味道很甜”
  • 句子 B：“苹果今天晚上发布财报，是一个短线投资机会”

这两句话中，“苹果”代表完全不同的含义。在句子 A 中，它是一种水果；在句子 B 中，它是一家科技公司。

在大模型中，当一个 Token 刚被嵌入到高维空间中时，它表示的是模型对这个词的统计语义认知，其中混合了该词在海量语料中出现过的各种语义模式。对于“苹果”这种多义词而言，其初始表示尚未完全确定当前语境下的具体含义。

那么，模型在当前的句子中该怎么理解它？这里就需要注意力机制了。注意力机制会根据相关性权重，对上下文中的 Value 向量进行加权汇总，并将汇总后的结果融合回当前 Token 的表示中。在“财报”语义下，“苹果”的科技、商业信息被放大，而水果、甜等属性被抑制。注意力机制通过融合上下文信息，完成了多义词在特定语境下的语义聚焦。

注意力的数学实现

我们从直觉上理解了“注意力就是看向周围的词”。那么计算机如何实现这个过程呢？

模型在处理一个词（Token）时，需要计算它与句子中其他词的关系，以赋予不同词不同的“注意力权重”。输入数据会通过线性变换，为每个词生成三个向量：

  • Q (Query)：词向量的主动检索表征。它编码了当前词想从上下文中提取哪些语法关系或语义特征。可以理解为当前 Token 对上下文的一种检索模式，它定义了该 Token 希望关注哪些特征，例如动词“吃”对应的 Query，训练后可能更容易匹配表示主语和宾语的 Key。
  • K (Key)：词向量的被检索特征编码。它定义了当前词暴露给其他词的“响应维度”，决定了它对外提供的语法或语义标签，例如“苹果”的 Key 可能编码了它所具备的语法和语义特征，使得其他 Token 能够判断它是否与当前查询相关。
  • V (Value)：词向量的实质性语义表征。它不参与关联度计算，而是在被其他 Token 关注时，向其提供实际需要传递的语义信息。

拆开来看这个公式：

第一步：算关联度（QKTQ K^TQKT）

拿“苹果”的查询向量 QQQ，去和句子里的所有词的键向量 KKK 进行点积运算，得到注意力分数。在几何空间中，方向越接近的向量通常会得到更高的点积得分，因此更容易被模型认为具有相关性。

多头注意力机制

GPT-3 实际实现时，使用了多头注意力机制。例如 GPT-3 的隐藏维度为 12,288。如果使用 96 个 Attention Head，那么每个 Head 的维度只有：12288/96=12812288 / 96 = 12812288/96=128，即每个 Head 只在 128 维空间中学习一类关系模式，最后再将 96 个 Head 的结果拼接回来。

为什么不直接使用一个超大的 Attention？

假设只有一个 Head，上下文长度为 n，那么 QKVQKVQKV 的维度都是 n×12288n \times 12288n×12288 维。理论上完全可以只使用一个 Head。Attention 直接在完整空间中计算相关性。

但语言中的相关性并不是单一概念。一个词可能同时涉及：

  • 语法关系
  • 语义关系
  • 指代关系
  • 位置关系

例如句子中的某个词，可能需要同时判断：

  • 它是谁的主语或宾语；
  • 它指代的是哪个实体；
  • 它与哪些词语具有语义关联；
  • 它与当前位置之间存在什么关系。

如果将所有关系都压缩到同一个 Attention 空间中，不同类型的关联关系容易相互干扰。因此 Transformer 将 Attention 拆分成多个 Head，让不同 Head 在不同子空间中学习不同类型的关系。

从直觉上看，一个 Token 的完整隐藏状态包含了丰富的信息，例如语义、语法、上下文和世界知识等，因此需要较大的隐藏维度来存储这些信息；但对于某一个具体的关系而言，例如主谓关系、指代关系或位置关系，通常只需要其中的一部分特征即可描述，并不需要完整的 12,288 维信息。因此：

  • 隐藏维度（Model Dimension）负责存储信息；
  • Head 维度（Head Dimension）负责提取关系。

Transformer 的设计并不是让每个 Attention Head 都理解整个世界，而是让不同 Head 从不同角度观察同一个世界。隐藏维度负责存储丰富的信息，多头机制负责从这些信息中提取不同类型的关联关系。正是这些关联关系不断修正和丰富 Token 的表示，最终让“苹果”从一个具有多种可能含义的词，逐渐演化为当前语境下准确而具体的语义表达。

从这个角度来看，Attention 并不是简单地“看向其他词”，而是在不断利用上下文重构 Token 的语义表示。Transformer 的强大之处，正是这种持续的语义聚焦与信息融合能力。
